搭建Blog基于Github pages & hexo

发表于 2016-04-21   |   分类于 Blog build   |  

缘起

还记得小时候第一次接触QQ，申请了自己的第一个QQ号，几经变迁，号码几何现在早已经起不清了。但是还清醒的记得那时熄灯之后在床上辗转反侧构想QQ的个性签名，终于有了一个想法，赶紧下床把在脑中的文字移到一张书签的背面。那似乎是我的第一次网络写作。

于是，从那一次的经历，到后来买黄钻装扮所谓的QQ空间。那些无关审美的非主流装饰，和矫情的文字，似乎也成我们这一代人的共同回忆。其实人们都希望拥有一个属于自己的空间，可以畅快的表达自己的情绪，或希望有人能看到，或只是写给自己。这也许也是建一个博客的想法一直占据在我的脑海中的原因。

促因

大一上学期接触了程序设计，大学的空闲时间也给自己更多机会去接触新鲜的工具。虽说

优秀的程序员，都是通过展示自己的代码牛逼来展示自己的牛逼的。二逼的程序员，通过展示自己的工具……

——vczh

但是由于天生对工具的喜爱，还是抓紧时间玩了很多工具，Pr Ps Id Keynote ……，都给自己带了一些成就感，也许这就是乱点科技树的快感。后来因为对Linux的兴趣加入Cugb Lug(一个QQ群)，结果Linux反正是至今一窍不通，但却在群里认识一帮牛逼的学长（比如公子, 范神, Tinko），学习到许多高端姿势。其中博客的字眼再一次出现在我眼中，进行了相关了解后，选择了github pages搭配hexo来搭建自己的博客，也就是现在大家看到的界面。

搭建过程

Github

选择Github的理由

  1. 立志成为码农的你已经或迟早要掌握Github
  2. Github pages提供了充分的可能性
  3. 简单——你看数学老师都会用 http://leochencipher.github.io
  4. 够装逼。。


https://guides.github.com/activities/hello-world/

So 我们开始吧

  1. 申请账号
  2. 点击右上角的加号”New repository”建一个叫“你的Github用户名.github.io”的repository（必须叫这个名字，一开始博哥和我说我还不服，其实是因为他讲不出理由，其实这是github pages规定的，新手还是照做吧）。
  3. 进入“你的Github用户名.github.io”里面的setting，在下面找到一个叫“Launch automatic page generator”的按钮，点进去，然后略。

至此，你的第一个github page就生成了，在地址栏输入你的Github用户名.github.io就可以访问了，上面的模版审美都还是过关的。其实在github pages上就可以直接生成链接更多的页面，以达到blog的效果，但是我并没有继续研究下去。好吧，接下去我们开始套模版了——hexo

Hexo

至于Hexo是什么，下面引用了其作者的介绍。

A fast, simple & powerful blog framework

没错它就是一套别人已经做好的模版，我们只要把自己的文章和一些个性化设置放在模版上，进而上传到github pages上，就可以简单的绕过不会网页设计的坑，做出赏心悦目的页面。

配置环境

这其实是我们刚刚没有配置github欠下的帐，因为Hexo等开源项目是放置在Github上的，所以我们需要将之下载到本地操作就需要借助Github的环境

  1. 安装Node.js环境
  2. 安装npm工具
  3. 安装Git以及SSH
  4. https://hexo.io/docs/index.html你懂的。

安装Hexo

npm install hexo-cli -g

好吧，可能以后真有人会看到，不能太坑。

npm install hexo-cli -g
hexo init blog 这里的blog可以设置成自己喜欢的名字，但是接下去的blog也要换掉，鉴于我并没有试过，还是别了吧
cd blog
npm install
hexo server

期间可能需要漫长的等待，比如几十分钟，但如果太久还是去检查网络环境，比如试着翻个Wall，也可以停掉再重新开始。反正到最后如果出现

INFO  Start processing
INFO  Hexo is running at http://0.0.0.0:4000/. Press Ctrl+C to stop.

就代表你成功了。这个过程中，截图一下安装目录信息，随时Google对你会有很大帮助，包治百病。现在，你可以在浏览器的地址栏输入http://0.0.0.0:4000/来访问hexo生成的供你临时预览的网页了。以后你会经常用到

hexo server

这个代码，它的作用是建造一个本地预览的网页，效果基本是实时刷新的，有些改动没有刷新时，你可以尝试用（Ctrl +C）停止server，用

hexo clean

清除缓存文件（这不会清除你对文件的改动），用

hexo server

再次启动本地预览，这时如果你的操作没有错误的话，改动都应该已经应用到本地预览的网页上了。

这个过程中，https://hexo.io ，会给你很大的帮助，我很不想提但是还是提一下吧，它可以选择中文版。

用Hexo写作

这个问题其实我也不想多提，因为在hexo的网页上也写的很清楚了，毕竟步骤也不多。主要分成直接添加文件到目录和用命令添加文件到目录两种方法。你可以不断尝试，预览，改正。

我想简单谈谈Markdown的问题。Markdown是一种轻量级的标记语言，它使我们可以不用学习html等比较难一点的标记语言就能进行很好的网页写作和排版。正如你所看到的，本文就是使用Markdown进行排版。

这里推荐你阅读https://www.zhihu.com/question/20409634这个问题的高票答案，其中推荐的一些介绍Markdown文档十分详尽，对比了它Html的区别，其实你如果不会Html的话直接略过所有有关Html的内容就好。github也对Markdown进行了介绍

在Mac OSX下我使用的是Mou这款编辑器。(习惯用sublime text的也可以选择安装markdown包来使用)

关于主题

如果你不喜欢Hexo的默认主题，那你需要先学习应用一些别人做的主题。我的Blog使用的是NexT这款主题，你也可以在Hexo的网页上寻找你喜欢的主题。具体应用的方法也很简单，推荐你用NexT学习一下主题的应用方法，因为它的介绍写的十分详尽。

最后一步——部署

当你在本地预览看到了没有错误的网页时，你就可以决定把它部署到Github完成我们的最后一步的时候了。

上传主要分两种方式

  1. 使用Github Desktop部署
  2. 使用命令行部署*

命令行式

这种方式需要进入你的hexo的根目录设置_config.yml文档，打开后找到deploy相关的设置，参考以下代码修改：

deploy:
    type: git
    repo: repository url \\去你的项目下复制
    branch: master
    message: \\留空，为默认
(所有冒号后面都要空一格)

接下来进入命令行，直接贴代码

hexo clean\\保险一点，清一下缓存
hexo generate\\在本地生成网页
hexo deploy\\将网页部署到Github等网站

参看https://hexo.io/zh-cn/docs/deployment.html

Github Desktop

先安装github desktop，参看https://help.github.com/desktop/guides/getting-started/,了解一下工作的流程。

  1. 将你的用户名.github.io下载到本地的github desktop上

  2. 在命令行中进行以下代码操作

    hexo clean\\同样为了保险
hexo generate\\在本地生成网页
  3. 这时候网页已经生成到了你的hexo的根目录里的public文件夹中，我们进入文件夹里把里面的所有文件复制到（1）中下载来的目录中
  4. 进入github desktop这时你应该看到软件上有提示我们刚刚的操作产生了一些changes，接下来我们只需要应用这些changes到github上就可以了。所以输入相关描述（这是是写给人看的不是写给机器看的），点击commit to master，再点击Sync同步之，成功。

值得一提的是我在用命令行式同步了几次后不知道操作错了什么，出现了各种错误，至今没有搞清楚，所以还是使用github desktop式更加通用一些。(ps: 已解决，只要deploy设置正确就没有问题，切记冒号后面有一个空格，空格后面直接填写信息不要乱加标点)
